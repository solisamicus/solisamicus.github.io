  • 无 close(c) 时：worker goroutine 阻塞在接收操作上，直到有新的数据或通道关闭。
  • 有 close(c) 时：当通道被关闭后，worker goroutine 仍然尝试接收数据，但因为通道已关闭且没有更多数据，<-c 会返回通道的零值（对于 int 是 0），导致不停地打印 Worker 0 received。
